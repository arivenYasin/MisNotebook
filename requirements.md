# 考研知识图谱复习系统 — 需求文档

## 一、产品概述

一个浏览器端网页应用，核心是一个交互式3D知识图谱。用户通过录入题目和知识点，逐步构建个人知识体系。系统根据做题记录动态追踪每个知识点的掌握程度和置信度，并智能推荐复习题目。数据持久化保存在本地（跨会话）。

## 二、数据结构

### 2.1 知识点（树形结构）

- 知识点组织为一棵多叉树，用户从空白开始自行添加。
- 所有题目只挂在叶子节点上。
- 如果用户需要将题目直接关联到非叶子节点（如"洛必达"本身），系统自动创建一个名为"XX本身"的子叶子节点来承载这些题目。
- 任何非叶子节点只有子节点，没有直接挂载的题目。

### 2.2 题目

每道题包含以下字段：

- **内容**：支持完整题干或简短标识（如"660第15题"），由用户自由选择。
- **关联知识点**：可关联多个叶子节点（可跨章节）。一道题关联多个知识点时，该题同时出现在所有关联知识点下。
- **难度**：用户自定义等级。
- **当前状态**：正确 / 错误。
- **若错误**：标记具体因为哪些知识点而不会。
- **做题历史**：记录每次做题的时间、结果、标记的失败知识点。

### 2.3 题目与知识点的多对多关系

- 一道题可以关联多个知识点。
- 当多道题同时关联相同的多个知识点时，这些知识点之间形成隐式关联，在图谱中体现为连线。

## 三、叶子节点状态模型

每个叶子节点独立维护以下状态：

### 3.1 掌握度 M

**计算方式**：当前错题数（该知识点下状态为"错误"的题目数量）通过 sigmoid 的负半轴映射到 0~1。

- 错题数 = 0 → M = 1.0（最高）
- 错题数越多 → M 越接近 0

**更新规则**：
- 录入一道新的错题 → 错题数 +1 → M 下降
- 做对一道之前错的题 → 错题数 -1 → M 上升
- 做对不会让 M 超过错题数为 0 时的值（正确不加正分，只消除负分）

### 3.2 置信度 C

**定义**：一个独立的 0~1 变量，反映"系统对当前掌握度判断的可信程度"。

**初始值**：C = 0（没有任何数据，完全不确信）。

**更新公式（每次做题时）**：

第一步——时间衰减：

```
C = C × (λ ^ (1 + k × F × M))^Δt
```

- λ = 衰减基础系数（如 0.95）
- Δt = 距上次做题的间隔步数
- F = 遗忘倾向（见 3.3）
- M = 当前掌握度
- k = 遗忘倾向调节系数（如 2~3）
- F × M 的作用：高掌握度且经常遗忘的知识点，置信度衰减更快；低掌握度时衰减速度不受遗忘倾向影响。

第二步——根据做题结果更新：

```
一致性分数：做对了 consistency = M，做错了 consistency = 1 - M
C = (1 - η) × C + η × consistency
```

- η = 学习率（如 0.4）
- 含义：结果符合当前掌握度的预期 → consistency 高 → 置信度上升。结果与预期矛盾（如高掌握度时做错）→ consistency 低 → 置信度大幅下降。

**注意**：计算一致性时使用的 M 是做题前的掌握度（更新掌握度之前的值）。

### 3.3 遗忘倾向 F

**定义**：该知识点最近50次做题中由对转错的次数 / 检查次数（最大为50）。

- 用于影响置信度的时间衰减速度，遗忘倾向越高说明这个知识点越容易忘记。

## 四、非叶子节点

非叶子节点不参与任何计算逻辑，仅用于可视化。

**可视化掌握度**：以该节点为根的整棵子树的所有错题数加总，过 sigmoid 得到掌握度颜色。

**不展示置信度**。

## 五、推荐算法

### 5.1 触发方式

用户点击"复习"按钮，系统生成一组题目。做完后可继续点击，可以一直做下去，最终目标是所有知识点都被掌握。

### 5.2 题目分配

每次生成的一组题目由三个来源按比例混合：

#### 来源一：记忆曲线到期的题目（刚性需求，优先安排）

- 之前做错的题，按间隔步数调度。
- 间隔步数（不是真实时间天数）：做对了间隔变长，做错了间隔缩短。
- 用类似 Leitner 的机制：做对往后移一步（间隔加大），做错退回第一步（间隔最小）。
- 到期的题必须出现。

#### 来源二：优先级驱动的练习题目（约 80% 的非到期配额）

使用优先级公式从叶子节点中选题：

```
priority = M × (1 - C) + w × M × (1 - M)
```

- 第一项 M × (1 - C)：捕捉"高掌握度但低置信度"——可能在遗忘，需要验证。
- 第二项 w × M × (1 - M)：捕捉"中间地带"——快要学会或刚开始忘的知识点，练习收益最高。M × (1 - M) 在 M=0.5 时最大值为 0.25，而第一项最大值为 1.0，因此 w=4 时两项对等。
- 掌握度很低（M 接近 0）的知识点两项都趋近于 0，自然排在后面，通过可视化和警告提醒用户去系统学习。

#### 来源三：探测性题目（约 20% 的非到期配额）

- 独立于优先级公式，用概率从置信度最低的知识点中随机选取。
- 不看掌握度，纯粹为了更新知识图谱的判断。
- 系统早期数据少时探测比例可以更高，数据充分后自动降低。

### 5.3 约束条件

- 刚做过的题有最小间隔保护，不会连续出现。
- 同一个知识点不应连续出很多题，要与其他知识点的题交错。
- 推荐的题目仅来自用户已录入的题库。

## 六、知识图谱可视化（核心界面）

### 6.1 整体形态

- 优先 3D 展示（可旋转、缩放、拖拽视角），2D 为备选。
- 知识点表现为大小不一的球体。
- 球体大小由下属题目数量或子节点数量决定。
- 球体颜色反映掌握度（如绿色→黄色→红色）。

### 6.2 层级与展开折叠

- 每一级可展开查看子节点，也可折叠为单个球体。
- 折叠时球体颜色反映整棵子树的汇总掌握度。
- 父子关系用明显的粗线连接，形成树状结构。

### 6.3 题目的表现

- 题目不作为独立节点出现。
- 仅关联单个知识点的题目被知识点球体"吸收"，体现为球的大小和颜色。
- 关联多个知识点的题目表现为知识点之间的连线。
- 少量共同题目 → 淡细线；大量共同题目 → 粗亮线。
- 题目连线远不如父子关系连线明显，除非共同题目非常多。

### 6.4 图上交互操作

- 点击知识点球体：展开/折叠、查看详情（掌握度、置信度、错题列表等）。
- 在图上直接添加子知识点（选择父节点 → 添加子节点）。
- 在图上直接添加题目（选择知识点 → 添加题目，支持关联多个知识点）。
- 编辑知识点名称、调整层级结构。

### 6.5 警告提示

- 掌握度非常低的知识点应有醒目的视觉警告（如红色高亮、闪烁、标记等），提醒用户需要系统学习。

## 七、复习流程

1. 用户点击"复习"按钮。
2. 系统按照推荐算法生成一组混合题目。
3. 逐题展示，用户标记：做对 / 做错（若错标记因哪些知识点不会）。
4. 每道题做完后实时更新相关知识点的掌握度、置信度、遗忘倾向。
5. 图谱实时更新颜色和连线。
6. 一组做完后可继续点击"复习"获取下一组。

## 八、技术要求

- 浏览器端网页应用（React）。
- 数据持久化保存在本地（使用 artifact 的 persistent storage API）。
- 3D 可视化（Three.js 或 d3-force-3d）。
- 不依赖后端服务器。

## 九、参数默认值（可调整）

| 参数 | 含义 | 默认值 |
|------|------|--------|
| λ | 置信度时间衰减基础系数 | 0.95 |
| η | 置信度学习率 | 0.4 |
| k | 遗忘倾向调节系数 | 2.5 |
| w | 优先级公式中两项的平衡系数 | 4 |
| 探测比例 | 非到期配额中探测题占比 | 20% |
| 每组题数 | 每次复习生成的题目数量 | 10 |
| 最小间隔 | 同一道题再次出现的最小间隔步数 | 3 |
